## 当前笔记生产工作流

### 步骤

1. **人工选图** → 放入 `内容生产/人工测试/[日期]笔记发布/`
2. **`/撰写笔记`**：读图分析 → 选模板 → 调库撰写初稿 → 自评输出终稿
3. **用户修改** → Claude Code 协助细改

### 图片处理工作流（博主素材图）

1. **人工粗分**（Python 工具）：封面候选 / 内页 / 可复用素材 / 待删除
2. **AI 细分**"可复用素材"：自动分入酒店客房 / 餐食 / 法国景点 / 意大利景点 / 瑞士景点
3. 一般博主图量正常，可跳过步骤1直接 AI 读文件夹写稿
4. **注意**：raaldew75 图量异常多，必须先人工粗分

## 笔记撰写规范

### 字数
**850-900 个中文字符**（含标点）。不足补真实场景细节，超出删冗余。

### 写作流程
1. 读图 → 产出**事实清单**（图里实际有什么、谁出镜、哪些服务细节可见）
2. 结合用户要求 + 风格方法论 + 库条目 → 初稿
3. 对照事实清单**逐句核查**（内容是否与图片一致？有没有编造？）
4. 输出终稿

### 常见错误（已踩过）
- **直接从库搬条目不核图**：场景库写"人少"，图片文件名是"人海全景"，完全相反。库是参考，图片是事实。
- **商务车真皮座**：每家都说的标配，没有区分度。应写具体方案（单段坐车不超过X小时、隔天半天轻松行程）。
- **无意义细节**："妈妈吃完第一块又指了第二块"——每个细节都要能回答「这说明了什么？」
- **结尾太含蓄**：先说以前什么状态，再说变化，结尾点明意义。不要让读者自己推断。
- **散文感**：短句为主，去行程日记感，偏小红书直说风格。做减法，不需要把所有卖点塞进去。

### 写作方法论完整结构
1. 开头（5选1）
2. 决策过程（必写，读者的镜子）
3. 信任建立（定制：定制师沟通过程；小团：人少/不跟陌生人/比大团好在哪）
4. 具体场景（含1-2个小插曲，瑕不掩瑜更可信）
5. 服务细节
6. 收尾（家人行为变化）+ 结尾钩子

## 受众分析方向

不建立个人对象档案。按5个人群分类收集笔记，**某类人群的笔记收集到20篇时，开启该人群的画像分析**：

| 赛道 | 说明 |
|------|------|
| 家庭定制 | 多人家庭，个性化行程 |
| 带妈妈出游（小团） | 子女带妈妈，小团拼团 |
| 带爸妈出游（小团） | 子女带父母双方，小团 |
| 和姐妹出游（小团） | 女性朋友结伴，小团 |
| 情侣度蜜月（小团或定制） | 蜜月/情侣，小团或定制均有 |

## 使用原则

- **先检索，再创作**：每次写作前先看场景库/金句库/标题库，复用已有素材
- **图片是事实，库是参考**：内容必须与图片一致，不能凭库里的内容编造
- **边用边沉淀**：`/分析笔记` 时自动提取金句/标题/场景补入对应库
