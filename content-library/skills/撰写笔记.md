# 撰写笔记

读取一组已选好的笔记图片，匹配模板，调用素材库，撰写笔记初稿。

## 使用场景

用户已人工选好一篇笔记的图片（封面 + 内页），放在一个文件夹里。

## 步骤

### 0. 建任务清单（强制，第一步）

调用此 skill 后，**立即用 TaskCreate 为以下每个步骤各建一个任务**，再开始执行。不得跳过。

| 任务名 | 说明 |

| 收集信息 | 向用户询问路径、价格、路线、人数、出行关系、业务类型 |
| 读图分析 | 读取全部图片，产出事实清单，与用户确认 |

### 1. 收集信息

询问用户：
- 图片文件夹路径
- 基本业务信息：**价格必须两个数字**（团费 + 含机票签证总花费）、路线、人数、出行关系（带父母 / 蜜月情侣 / 带孩子等）、**业务类型**（小团 / 定制）
- 有无特别想突出的卖点或场景（可选）

### 2. 读取图片，产出分析文档

读取所有图片（封面优先），产出一份**读图分析**，内容包含：

**图片事实清单：**
- 每张图确认的事实（地点、人物、天气、可见细节）
- 特别标注：哪些图只有风景无人物 / 哪些图有具体人物表情 / 哪些内容不能从图片确认

**用户要求汇总：**
- 出行关系、路线、价格、人数、业务类型
- 特殊情况（腰不好 / 第一次出国等）
- 字数、模板等写作要求

将此分析文档与用户确认后，再进入下一步。

### 3. 选择模板

读完图片后，**向用户给出模板建议并说明理由，由用户确认使用哪个模板**后再进入下一步。

> 当前模板文件：`素材库/笔记模板库/小团笔记模板.md`，包含A/A2/B/C/D/E六种结构类型。

**模板选择参考（按图片内容判断，给出1-2个建议选项）：**

| 图片信号 | 建议模板 | 理由 |
|---------|---------|------|
| 多位家庭成员各有独立镜头（老人、孩子、大人分开拍） | D型 | 按成员满意点分项，结构自然 |
| 照片丰富、按地点逐天展开，风景/打卡照为主 | B型 | 图片直接支撑行程叙述 |
| 酒店、餐食、交通等服务细节照片明显 | **仿写**（优先）或 E型 | 仿写：有机结构，从真实关切出发；E型：服务维度公式分项，慎用 |
| 情侣/人物照为主，图片角色次要，重点想讲选团故事 | A/A2型 | 70%文字叙事，图片只作辅助 |
| 已回总结感强，图片以体验场景为主 | C型 | 体验总结+顺带选团建议 |

> **仿写 vs E型选择原则**：图片是景点+服务混合、找不到清晰的"几位家庭成员独立满意点"时，优先选仿写而非E型。E型容易写成服务类别清单，仿写从博主真实担忧出发，结构更有机、评论率更高。仿写路径见下方「仿写模板库」行。

> A/A2 区别：A型叙事感更强（有淘汰剧情感）；A2型更清单式（适合干货风）。可同时推荐两个，用户选其一。

**小团 vs 定制写法区分（确认模板后，按业务类型调整强调点）：**
- **小团**：强调「人少/就我们这几个人/不跟陌生人/比大团好太多/比定制便宜」，人数是核心卖点
- **定制**：强调「行程完全按我们需求来/提前问过我们的偏好/出了问题有专人处理」，个性化是核心卖点

### 4. 按锚点调库，撰写初稿

**动笔前：先读一遍源笔记，提取节奏**

选完模板后，找到对应的源笔记完整读一遍。**目标是提取写作节奏，不是复制业务内容。**

- **要提取的**：① **写作节奏**：开头用几句话/什么句式、每段末尾怎么收束（有没有切换视角的杀手句）、情感弧度怎么建立、结尾怎么喊话；② **服务维度**：每个人物/段落覆盖了哪几个服务维度——记下来，作为本篇内容选择的参考基准。
- **不照搬的**：定制自由行笔记里的路书App/公共交通/远程顾问——这些是不同业务DNA，照搬后整篇语感都会偏。E型转换规则见下方⚠️提示。

**服务维度选择逻辑（所有模板适用，动笔前确认）：**

每篇笔记写哪些服务维度，从以下两个来源取，优先级并列：
1. **源笔记维度**：原笔记写了哪些维度，本篇优先参考。能成为模板，说明这些维度已被验证。
2. **人群特征**：这类出行者核心关切是什么（带孩子 → 讲解/节奏/餐饮；带父母 → 体力/住宿/行程松紧；蜜月 → 两人私密时间/浪漫体验）。

> **目标读者 = 笔记主角的镜像**：写的是一家三口，目标读者就是有类似情况的家庭。维度贴合主角需求，自然贴合目标读者，不需要另外推导。

**通用服务维度参考表（按人群相关性选取，不要全写）：**

| 维度 | 说明 | 最相关人群 |
|------|------|-----------|
| 定制师前期沟通 | 需求理解、方案调整、个性化安排 | 所有人群 |
| 官导讲解 | 大景点预约讲解，孩子/老人的接受度 | 带孩子/带父母 |
| 行程节奏 | 松紧是否匹配这类人群 | 带孩子/带父母 |
| 司导服务 | 出行中贴身执行，照顾人的细节 | 所有人群 |
| 餐饮推荐 | 定制师/司导推荐，踩准口味 | 带孩子/蜜月 |
| 酒店选址/类型 | 是否匹配需求（别墅/市中心/景区内） | 所有人群 |
| 0购物+纯玩 | 信任背书 | 所有人群 |
| 签证/机票/交通全包 | 省心度证明 | 所有人群 |
| 突发情况处理 | 出了问题如何解决 | 视情况 |

读完后，按自己的内容重新写，不复用原文句子。

**动笔前：用一句话确定本篇核心价值**

这篇笔记在说什么——不是"记录了一次旅行"，而是给目标读者一个什么核心感受。
> 例：省心（把一切交给专业的人）/ 安心（有老有小出国不用操心）/ 开眼界（孩子第一次出国）/ 浪漫（蜜月每个环节都被想到了）
把这句话写下来，带着它动笔。每写完一段问：这段在支撑这个核心价值吗？回答不了的段落，是废话或者方向跑偏了。

**⚠️ 写稿前先确认：图片在本模板中的角色**

不同模板，图片的作用完全不同。写稿的内容主体来源于模板骨架和素材库，图片只在各自对应的范围内使用：

| 模板 | 图片的作用 | 正文主要内容来源 |
|------|-----------|----------------|
| **仿写（有机型）** | **读图发散**：每张图能启发写什么？先列灵感，再对照参考笔记的维度选取，取交集。不是每张图都要写，也不要硬凑到每个维度。**图片是创作起点，不是创作边界**：可以根据图片延伸周边场景（如广场有人跳舞、山顶泡面的价格梗等）。初稿只写从图片能观察到的内容，延伸场景由用户在看到初稿后补充修改。 | 参考笔记节奏 + 用户提供的业务信息 + 图片触发的具体场景。结构由真实关切决定，不由图片和服务类别决定 |
| **A/A2（决策型）** | 确认人物关系；提供1-2个具体场景细节穿插佐证 | **选团决策故事**（淘汰过程/测靠谱度/服务细节）。图片中的景点不逐一展开描写，仅作为"选对了"的侧面佐证偶尔提及 |
| **B（行程日记型）** | 直接支撑行程场景描写 | 图片+场景库，图片是主要内容来源 |
| **C（体验总结型）** | 支撑体验感受描写 | 体验部分来自图片，选团建议部分来自库 |
| **D（家庭成员视角型）** | 找各成员满意瞬间的证据 | 按家庭成员分项；每个成员的满意维度由**人群特征+源笔记维度**决定，不由图片景点决定；每个维度内容 = 需求→服务→超出预期 |
| **E（服务维度分项型）** | 找各服务维度的佐证 | 按服务类别分项；每个维度由**人群特征+源笔记维度**决定，不由图片中的景点决定 |

> **【通用原则】所有模板的内容维度均由人群特征+源笔记维度决定，不由图片决定。图片是写作起点或佐证，不决定写哪些维度、写什么内容。**
> 选了A/A2型：**克制图片描写**。整篇约70%写选团故事，图片景点最多穿插1-2处，其余不提。
> 选了D/E型：**图片不决定结构**。先按人物/服务维度搭好骨架，再回头找图片里对应的证据填入。
> 选了D/E型：**每个维度必须有一个具体执行动作作为证明，不能只写结论。**「行程比较松」是结论；「中午送爸妈回酒店休息，下午再接」才是证明。写完每个维度后问自己：读者看完知道这个问题是怎么被解决的吗？

**初稿完成后，对照读图分析核查（事实层面）：**
- 有无编造图片里没有的具体行为、对话或反应？（A/A2型：选团故事部分不需要图片支撑，只需核查穿插的景点细节）
- 出镜人物是否写对？（谁在拍、谁在被拍、有没有合照）
- 库条目如果涉及具体场景，是否有图片对应？（如写"展厅人少"，图片是否确实人少）

核查通过后，依次完成**自评 → 对标源笔记 → 修改**，完成后进入步骤5统一输出。

**自评清单（输出前逐条过，写作质量层面）：**
1. 有没有废话？（能回答"这说明了什么"才留，回答不了就删）
2. 场景角度是否全在重复同一类？根据本篇人物关系和用户要求判断，确认角度有变化（情绪 / 里程碑 / 意外惊喜 / 行程灵活 / 小插曲等）
3. 有没有1-2个小插曲？（**A/A2型不强制**。其余模板：一路夸不真实，小遗憾反而增加可信度和评论互动）
4. 句式是否单一？（多个场景连续用同一结构就是失败，要混入反转、感受、行为细节等）
5. 字数是否在目标区间（850-900字）？

**对标源笔记（自评后执行）：**

逐段对比初稿与源笔记，分框架和内容两个维度：

- **框架层面**：每个段落是否对齐？有无段落缺失、顺序错位、或两段内容揉在一起？
- **内容层面**：逐段说明比原文好在哪里、差在哪里。差的部分列出具体修改方向。

对比完成后，**用1-3行说明主要差在哪、改了什么**，修改完成后进入步骤5统一输出。不需要输出逐段对比过程。

### 写作原则（每次撰写都适用）

- **具体场景 + 感受评价，不能只有其中一个**：只写感受没有画面（「导游很贴心」）→ 读者没有记忆点；只写场景没有落点有时太干。正确做法：具体场景给读者看见，感受/评价帮读者定性，顺序灵活调整。例：「司导随身带折叠小凳，看老人累了就拿出来——这种细节大团根本顾不到」；「妈妈在站台站了很久不愿意走，这是她第一次看见雪山」
- **必须有情绪弧度**：出发前的担心/惶恐（真实写出来）→ 一个具体转折场景（不是笼统「超出预期」）→ 结尾用家人行为变化收束
- **可信度靠具体数字**：人均X万、X人一辆车、X天行程，比模糊描述更有说服力。**咨询了多少家旅行社（如「挑了8家」）可自行编造一个合理数字，不需要问用户。**
- **可以有1-2个小插曲**：轻描淡写一个小不足，比一路夸更真实，评论互动也更多

### 常见错误预防（每次撰写前过一遍）

**1. 读图先确认出镜人物**

读图第一步：图片里出镜的是谁？是用户本人还是被拍对象？有没有合照？
不确认清楚，后续所有场景描写都会出错。
> 案例：3.29笔记全部图片出镜均为妈妈，女儿在镜头后，没有合照。误认后导致多处描写失准。

**2. 库条目用前必须对图确认**

从库取用任何条目，必须在事实清单里找到对应图片支撑，才能写进稿子。找不到图片的库条目一律不用。
> 案例：从场景库搬了「蒙娜丽莎展厅人少」，但实际图片文件名是「人海全景」，完全相反。

**3. 服务/票务的归属要写清楚**

票、服务、安排的提供方写清楚——旅行社包含的，写「旅行社提前买好了」；不要写「买了」，读者会误以为是自费项目。
> 案例：「买了屋顶票上去」→ 改为「旅行社提前买好了屋顶票」。

**5. 小插曲三步结构**

① 问题出现（轻描淡写）→ ② 主角的反应（不抱怨/顺其自然）→ ③ 结果还是好的（收束）
> 案例：米兰下雨（问题）→ 妈妈穿粉裤子没抱怨（反应）→ 屋顶票日落值了（收束）

**6. 场景具体靠位置，不靠名称**

不用品牌英文名、画作名让读者去查资料，用位置做锚点：
- 好：佛罗伦萨中央市场一楼的三明治 / 许愿池对面街角的冰激凌
- 差：All'Antico Vinaio的三明治 / 拉奥孔群雕正面角度

**7. 句式和语义不能平，要有变化**

多个场景连续出现，最忌每句都是「地点 + 人物怎么了」的固定结构。每篇至少混入3种不同角度：里程碑（第一次看见雪山）、意外反转（担心高反倒是我先喘了）、情绪/感受（安静站了很久不走）、小插曲（下雨但结果值了）、行为细节（导游没催/吃干净了）。

### 5. 排版并统一输出

先对终稿执行排版检查：

- 删除句子中间因终端换行产生的多余空格（如「导游 随身带」→「导游随身带」）
- 段与段之间保留一个空行，不多不少
- 每行末尾不留空格
- 标点符号前后不加空格
- 数字和汉字之间不加空格（如「8 家」→「8家」）

排版完成后，一次性输出以下全部内容，不附任何说明文字：

### 7. 修改过程中 → 经验沉淀

用户提修改意见时实时识别类型，稿子确认后一次性写入：

- **行业常识**（用户说的是业务事实）→ 追加到本文件「行业常识」区，格式：`- [事实]（来源：日期）`
- **写法指导**（用户说"这句改成…"）→ 对比 AI 原句和用户写法，提炼规律，追加到「写作风格积累」区，格式：`AI原句 → 用户写法 → 规律`

## 行业常识

写稿前需了解的业务事实，避免从错误假设出发：

- **司导 vs 具名导游**：测靠谱度/对话阶段，客人尚未确认出行，旅行社不会敲定具体导游姓名，只能说「司导」。明确出行后才叫「王导」等具名人物。（来源：2026-04-10）
- **波西塔诺酒店价格**：波西塔诺的酒店价格远高于阿马尔菲其他地方，客人需求不宜写「想在波西塔诺多留一晚」，应写「想在南意多住两晚」。（来源：2026-04-10）
- **旅游平台**：写笔记时用小红书、某程、某猪（飞猪）。某蜂窝市场份额已很小，不必提及。（来源：2026-04-10）
- **定制师与司导的分工（所有模板适用）**：定制师负责出发前——理解需求、设计路线、安排酒店、预定门票（定制师预定，不是司导），并将客人需求和注意点提前告知司导。司导负责出行中贴身执行和服务。出行中问题处理分工：**临时现场问题**（路况/景点临时关闭等）→ 司导现场直接解决；**后台事务**（新增景点门票/加一晚酒店/更换司导等）→ 定制师远程处理，我们也有24H支持。**推荐餐厅**：定制师出发前提供推荐清单，司导现场也会推荐，两者都有。**写笔记时：出发前的规划/沟通写定制师；出发后的现场服务写司导；出行中出了问题可以写「司导/定制师马上处理」按实际情况分配。**（来源：2026-05-01更新：2026-05-02）
- **少女峰的交通方式**：上少女峰分两段——第一段是大缆车（Eiger Express，格林德瓦终点站→冰河站），第二段是登山齿轨火车（冰河站→少女峰顶）。写稿时如不确定具体段次，不要强行写交通方式，留空或只写地点名即可。（来源：2026-05-20）
- **写稿用具名人物，不用职称通称**：笔记里的定制师和司导，如果用户提供了名字（如「图图」「王导」），全程用名字，不用「定制师」「司导」作通称。具名人物比职称通称真实感强得多，是素人笔记信任感的重要来源。用户没提供名字时保持通称，不要自行编造。（来源：2026-05-21）

## 写作风格积累

**1. 不用破折号（——）**
AI原句：「吃完也只有一句话——这辈子吃过最好的」
→ 用户写法：「那不勒斯那张披萨排了快半小时队，但是我这辈子吃过最好吃的。」
→ 规律：破折号是书面写法，素人分享用句号/逗号代替。

**2. 食物描写靠行为反应，不靠感官堆砌**
AI原句：「汤汁挂得很浓，鲜味很直接，两个人没说话，各自埋头吃完」
→ 用户写法：「佛罗伦萨那顿龙虾意面是真的鲜，到现在还惦记着；吃到一半我问老公好不好吃，他没抬头」
→ 规律：食物好不好用人的反应说（还惦记着/没抬头），而非堆砌感官形容词（浓/鲜/香）。过度具体反而显假。

**3. 旅行社专业回答需具体到地点+时间**
AI原句：「有，我知道一个悬崖上的位置，黄昏时光从海面来」
→ 改后：「可以安排，波西塔诺山腰有一个悬崖观景台，站上去整个村子在脚下，傍晚六点前后光从海面来」
→ 规律：展示专业度靠具体信息（地点名+时间），而非泛泛的「有」。

**4. 竞争对手要有一定专业水准，但不如己方精准**
AI原句：「到了当地让司机问一下就行，波西塔诺很多地方都能拍」（太草包）
→ 改后：「南意沿海都有景，波西塔诺或者阿马尔菲都行，到时候让司机带你们转转」
→ 规律：竞对应知道大方向（正确地区），缺乏具体执行（无位置/时间）。不能写成毫无水准。

**5. 酒店描写：核心场景+人的反应，不堆砌装修细节**
AI原句：「金色水晶吊灯，黄金色的床头，有点夸张，但住进去就懂为什么了」
→ 用户写法：「推开门就是地中海。早上在露台吃早餐的时候，眼前就是一整片蓝。那天我的早餐吃得格外的慢。」
→ 规律：装修细节（吊灯/床头）无感染力；一个核心场景+人的行为反应（早餐吃得格外慢）更真实有力。

**6. 路线描写用白话，不用官方归纳词**
AI原句：「从北边多洛米蒂的湖光山色……雪山湖泊、运河古城、海景悬崖」
→ 改后：「行程是从北往南开的。先在多洛米蒂的湖边转了一圈，山峰扎进云里，湖是那种很浓的绿；然后威尼斯、佛罗伦萨、罗马，每个地方住够了再走；最后到南意，停下来慢慢住。」
→ 规律：「湖光山色」「运河古城」是官方归纳词，读者无画面感。用白话（住够了再走/停下来慢慢住）+ 景物感官细节（山峰扎进云里/湖是那种很浓的绿）替代。

**7. 用户语音输入是素材方向，不是定稿**
用户语音描述 → AI直接放进笔记 → 用户反馈：「你好像把我的话完完全全地放到了笔记中，我是用语音说的，不是斟酌过的打字，需要你润色。」
→ 规律：用户用语音/口述给出的修改意见是方向和素材，AI需要将其转化为符合小红书风格的语言（精练、自然、短句）。不能原句照搬，哪怕方向是对的。（来源：2026-05-20）

**8. 图片里的服装颜色/外貌细节不写，行为场景才写**
AI原句：「她穿了件黄裙子，在登机廊桥回头让我拍照」→ 直接描述图片画面
→ 改后：「在登机廊桥上，我喊了一声妈妈，她回头看见我举着手机，马上笑得合不拢嘴」→ 行为+反应
→ 规律：图片里的服装颜色、配件等外貌细节是视觉信息，写出来回答不了「这说明了什么」；行为场景（做了什么、什么反应）有叙事意义才值得写。写图片内容前先问「这说明了什么？」，回答不了就不写。（来源：2026-05-20）

**9. D型开头用具体人设，不用泛化身份**
AI原句：「来自一个普通家庭的真实分享」（类别标签，无角色感）
→ 用户写法：「来自纯素人老母亲滴真实分享」（具体人物角色，立刻建立视角）
→ 规律：D型笔记开头的"我是谁"要给读者一个具体角色（老母亲/爸爸/姐妹），而不是一个分类标签（普通家庭/普通人）。"老母亲"比"普通家庭"多了角色感，读者立刻知道这篇笔记从谁的视角说话。（来源：2026-05-21）

## 封面判断标准

- 背景是知名景点，清晰锐利
- 人物比例合适，正脸/侧脸有表情
- 朴素自然感（无过度修图，不像商业摄影）
- 同一景点不同姿势可全部推荐，发多篇笔记去重
