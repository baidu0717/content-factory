# 仿写笔记

对指定源笔记进行仿写/洗稿，融合图片内容和用户提供的卖点，独立于「撰写笔记」skill。

## 使用场景

已有一篇目标笔记（源笔记），在保持其段落结构和写作节奏的基础上，替换业务内容，融入本次出行的图片场景和用户想要突出的卖点。

---

## 步骤

### 0. 建任务清单（强制，第一步）

调用此 skill 后，立即用 TaskCreate 建以下任务：

| 任务名 | 说明 |
|--------|------|
| 收集信息 | 基本信息、客户背景、各成员卖点、源笔记路径 |
| 图片声明 | 用户说明每张图的用途 |
| 读源笔记提取结构 | 段落收尾方式、情感结果位置、写作节奏 |
| 洗稿 | 按源笔记结构填入新内容 |
| 输出 | 终稿 + 标题 + 封面建议 |

---

### 1. 收集信息

向用户询问以下内容，一次性问完：

**基本业务信息：**
- 图片文件夹路径
- 路线（法意瑞 / 意大利 / 法国等）
- 人数 + 出行关系（一家几口、带父母、蜜月等）
- 团费 + 含机票签证总费用（两个数字都要）
- 业务类型（定制 / 小团）
- 源笔记路径（用户指定，AI 不推荐）

**客户背景（AI 读图读不出来的信息）：**
- 有无特殊情况：身体限制、第一次出国、特殊安排等
- 本次行程有什么特别背景需要交代

**各成员卖点（核心，动笔前必须收集）：**
> 每个家庭成员，你最想写的那一点是什么？一句话即可。没有想法的成员说「按源笔记来」。

---

### 2. 图片声明

读取图片文件夹，列出全部图片，请用户说明：
- 哪张图想写什么内容（一句话）
- 哪张图只是证明去了，不需要写进正文

**AI 不主动从图片提取卖点。用户声明了什么，AI 才写什么。**

---

### 3. 读源笔记，提取结构

读取用户指定的源笔记，内部提取以下内容：

1. **段落结构**：有哪些分项，分别对应谁
2. **每个段落如何收尾**：当下具体结果 / 行为变化 / 功能性收束？
3. **情感结果放在哪里**：各分项内 / 统一在最后一段
4. **写作节奏**：开头怎么进、各段怎么过渡、结尾怎么收

提取完成后，只输出一行结构摘要供用户确认，按源笔记实际结构描述，例如：
> 「5个✅分项（我/老公/儿子/女儿/爸妈），各段功能性收尾，情感结果统一在最后一段。」
> 「选团决策叙事结构，淘汰过程→测靠谱→服务细节，结尾转化钩子。」

用户确认无误后进入洗稿，无需逐条审核。

---

### 4. 洗稿

按源笔记段落结构逐一替换内容：

**替换内容：**
- 家庭成员 → 本次出行的实际成员
- 各成员卖点 → 步骤 1 用户提供的内容
- 图片场景 → 步骤 2 用户声明的图片点（只写声明的，其余不写）
- 价格、路线、天数 → 用户提供的数字

**结构约束：**
- 每个段落的收尾方式，严格照源笔记的模式，不自己发明新的收尾
- 情感结果放的位置，照源笔记来，不提前或后移
- 不因为「我们多一个成员」就改变段落收尾的规律

**写作约束：**
- 每个场景必须有落点：这说明了什么？回答不了就删
- 结论直接说，行为作为佐证——不要让读者自己推断（好吃就直接说好吃）
- 字数：850-900 字

**洗稿完成后自评：**
1. 每个段落收尾是否和源笔记同类型？
2. 情感结果是否放在源笔记对应的位置？
3. 用户声明的图片内容点是否都写进去了？
4. 有没有编造用户未声明的内容？
5. 字数是否在目标区间？

---

### 5. 输出

排版检查后，一次性输出：

- 删除句子中间因换行产生的多余空格
- 段与段之间保留一个空行
- 数字和汉字之间不加空格
