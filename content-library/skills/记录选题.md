# 记录选题

## 能力描述

当用户有一个内容想法时，快速记录到选题库，避免灵感流失。这是内容生产的第一步。

## 触发词

- 「记录选题」
- 「记录想法」
- 「添加选题」

## 执行流程

### 1. 收集信息

询问用户：
- 选题内容（核心想法）
- 目标对象（如果明确）
- 灵感来源（可选）

### 2. 记录到选题库

在 `01-内容生产/选题管理/00-选题记录.md` 中添加记录：

格式：
```markdown
## [记录时间] - [选题标题]

**选题内容**：
[详细描述]

**目标对象**：
[目标人群描述]

**灵感来源**：
[灵感来源]

**状态**：待深化

---
```

### 3. 初步分析

如果选题明确，我会：
1. 快速检索素材库，看是否有相关内容
2. 提供初步建议：
   - 是否可以复用已有内容？
   - 是否需要新建对象分析？
   - 建议的创作方向

### 4. 确认完成

告诉用户：
- 选题已记录
- 初步分析结果（如有）
- 下一步建议

## 示例对话

**用户**：记录选题

**AI**：请告诉我你的选题想法：
1. 核心内容是什么？
2. 目标对象是谁？（如果明确）

**用户**：我想写一篇关于大学生如何通过小红书赚到第一桶金的内容，目标对象是在校大学生。

**AI**：
已记录选题到 `01-内容生产/选题管理/00-选题记录.md`

**初步分析**：
- 检索素材库：找到 2 篇相关的已发布笔记
- 对象库：已有「大学生-副业赚钱」的对象分析
- 建议：可以复用对象分析，结合已发布笔记的成功经验，创作新内容

**下一步建议**：
说「分析对象：大学生」，我会基于对象分析和素材库，帮你生成文案。

## 选题记录格式

在 `00-选题记录.md` 中，选题按时间倒序排列，每个选题包含：

```markdown
## 2026-02-08 14:30 - 大学生如何通过小红书赚到第一桶金

**选题内容**：
分享大学生在小红书上变现的具体方法，包括选品、内容创作、流量获取、变现路径等，用真实案例和数据说话。

**目标对象**：
在校大学生，想通过互联网赚钱，但不知道从哪里开始。

**灵感来源**：
看到后台有很多大学生咨询如何在小红书赚钱。

**状态**：待深化

**初步分析**：
- 相关素材：2 篇已发布笔记
- 对象分析：已有「大学生-副业赚钱」
- 建议方向：案例驱动 + 具体步骤

---
```

## 选题状态说明

- **待深化**：刚记录的想法，还没开始创作
- **创作中**：已移动到 `02-创作中的选题/`
- **已发布**：已移动到 `03-已发布的笔记/`
- **暂缓**：暂时不做，但保留想法

## 注意事项

1. **快速记录**：不要过度思考，先记下来再说
2. **保留原始想法**：记录时保持想法的原始性，不要过度加工
3. **时间戳**：每个选题必须有记录时间
4. **初步分析**：如果能快速检索到相关素材，要提供初步建议
5. **状态管理**：选题状态要及时更新

## 关联能力

- **检索素材**：记录选题后，会自动检索相关素材
- **分析对象**：如果选题明确了目标对象，会建议进行对象分析
- **生成文案**：选题深化后，会进入文案生成流程
