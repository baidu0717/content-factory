# 推荐话题

根据笔记内容推荐小红书话题标签，提高精准曝光。

## 触发词

- 「推荐话题」
- 「推荐标签」
- 「用什么标签」

## 执行流程

### 1. 分析笔记内容

提取：
- 目的地（法意瑞 / 西班牙 / 冰岛等）
- 出行类型（小团 / 定制）
- 人物关系（带妈妈 / 带爸妈 / 蜜月情侣 / 家庭 / 带孩子 / 姐妹出游等）
- 有无特定景点（少女峰 / 五渔村 / 罗马等）

### 2. 检索话题标签库

读取：`content-library/内容生产/素材库/话题标签库.md`

**所有推荐的标签必须来自该库，不得自行编造未收录的标签。**

> 理由：小红书话题必须是平台上真实存在、有人发布过内容的话题，自行编造的话题不会有流量。

### 3. 选取标签

**优先使用推荐组合（快捷路径）：**

查看话题标签库「五、各人群推荐组合」，如果笔记的出行关系和业务类型与某个组合匹配，直接使用该组合，无需逐一筛选。

**如无现成组合，按以下维度逐一选取：**

- **目的地标签**（必选1-2个）：从「一、目的地标签」选；有对应景点图片时加景点级标签
- **出行类型标签**（必选1个）：从「二、出行类型标签」选；小团用小团标签，定制用定制标签
- **人群标签**（必选1个）：从「三、人群标签」按出行关系选
- **通用标签**（选1-2个）：从「四、通用标签」选高频通用标签

如果话题标签库中没有合适的标签，直接告知用户，**不要推荐未收录的标签**。

### 4. 搭配原则

- 总数：5-7个
- 结构：目的地（1-2）+ 出行类型（1）+ 人群（1）+ 通用（1-2）
- 标签必须与内容高度相关，不蹭无关热门

### 5. 输出格式

```
推荐组合：#xx #xx #xx #xx #xx #xx

选取说明：
- 目的地：#xx（理由）
- 出行类型：#xx（理由）
- 人群：#xx（理由）
- 通用：#xx #xx
```

## 注意事项

1. **只用话题标签库里已收录的标签**，未收录的一律不推荐
2. 目的地 + 出行类型 + 人群三个维度必须覆盖
3. 每次发布后记录哪个标签带来的流量更多，及时更新标签库
