# 记录素材

将爆款笔记收录进 Obsidian 素材库，通过飞书表格 → 脚本自动化完成。

## 完整流程

```
发现爆款笔记
  → 填入飞书表格（收录状态=待收录）
    → 运行收录脚本
      → 笔记自动出现在 Obsidian 素材库
        → 自动触发 /分析笔记
```

---

## 第一步：填写飞书表格

在对应表格中新增一行，填写以下字段：

### 必填
| 字段 | 说明 |
|------|------|
| 标题 | 笔记原标题 |
| 正文 | 笔记完整正文 |
| 话题标签 | 原帖标签，空格或逗号分隔 |
| 作者昵称 | 发布者昵称 |
| 发布时间 | 格式 YYYY-MM-DD |
| 笔记链接 | 小红书原帖链接 |
| 点赞数 / 收藏数 / 评论数 | 当前数据 |
| 封面 | 封面图片附件 |
| 小团/定制 | 多选：小团 / 定制 / 定制自由行 / 同行小团 / 亚宁写的 / 真实客人反馈 |
| **收录状态** | 设为**待收录** |

### 选填
| 字段 | 说明 |
|------|------|
| 图片2 / 后续图片 | 其余内页图片附件 |
| 浏览数 | 如能获取 |
| 内容类型 | 多选标注 |
| 账号属性 | 个人号 / kos号 |
| 备注 | 收录原因、特别亮点等 |

### 五张飞书表格
| 表格 | 用途 |
|------|------|
| 法意瑞 | 法意瑞目的地笔记 |
| 西班牙 | 西班牙目的地笔记 |
| 意大利 | 意大利目的地笔记 |
| 行程大师 | 行程大师相关笔记 |
| 聚光投放 | 聚光投放相关笔记 |

---

## 第二步：运行收录脚本

填写完成后，在项目根目录运行：

```bash
npx ts-node scripts/collect-to-obsidian.ts
```

脚本会自动：
1. 读取所有表格中「收录状态 = 待收录」的记录
2. 下载图片到本地
3. 生成 `笔记.md`（含元数据、互动数据、图片嵌入、正文）
4. 将收录状态更新为「已收录」

---

## 收录结果

笔记根据「小团/定制」字段自动分配到对应素材库：

| 标签 | 存入位置 |
|------|---------|
| 小团 / 定制 | 定制和小团爆款笔记库 |
| 定制自由行 | 定制自由行爆款笔记库 |
| 同行小团 | 同行小团爆款笔记库 |
| 亚宁写的 | 亚宁写的笔记库 |
| 真实客人反馈 | 真实客人反馈笔记库 |

文件夹命名格式：`YYYYMMDD-笔记标题`

---

## 第三步：自动触发分析笔记

收录脚本运行完成后，**立即对每篇新收录的笔记执行 `/分析笔记`**：

1. 告诉我哪个笔记库有新收录（或直接说「分析新收录的笔记」）
2. 我会找到所有还没有 `笔记分析.md` 的文件夹，逐一执行分析
3. 分析结果自动写入 `笔记分析.md`，并同步更新场景库、金句库

> 如果一次收录了多篇，可以说「批量分析」，我会逐篇处理。
