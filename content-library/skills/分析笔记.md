## 第二步：读取内容

1. 读取 `笔记.md`，获取：互动数据、正文、话题标签、内容类型、**备注**
2. 读取图片：封面.jpg → 图2.jpg → 图3.jpg（最多3张）
3. 读取模板库：
   - `content-library/内容生产/素材库/笔记模板库/小团笔记模板.md`（用于结构对比）
   - `content-library/内容生产/素材库/笔记模板库/仿写模板库.md`（用于判断是否符合入库或已有相似参考）

**备注处理规则**（在分析开始前先判断）：
- 备注中含「模板」字样 → 分析完成后**直接将此笔记写入模板库**，无需询问
- 备注中含正面评价（「可以参考」「不错」「好」等）但未提模板 → 分析完成后**询问用户**是否纳入模板库
- 备注中含「定制自由行」「不是我们业务」等说明 → 在分析开头注明，内容仍正常归库（开头/结尾/场景等均可参考，发布时再视业务调整）
- 无备注 → 按正常流程走，结构匹配环节再判断

## 第三步：生成分析

按以下结构输出，每个部分都有明确规则。

---

### 零、采集备注（如有）

直接引用 `笔记.md` 中的备注原文，并标注对本次分析的影响：

> 备注原文：「[备注内容]」
> 参考意义：[这条备注对分析的影响，如：用户认为结构可参考 / 业务不匹配仅作结构参考 / 已标记为模板]

### 一、封面分析

- 背景景点（尽量具体）
- 人物占比（0% / 约10% / 约30% / 约50% / 约70%）
- 人物情况（几人、关系、姿态）
- 构图特点

**朴素自然判断**：如封面有无滤镜感、光线真实、像随手拍，用 ⭐ 重点标注：
> ⭐ **封面风格朴素自然**——[为什么朴素]，这类封面爆率高，值得关注

### 二、图2 / 图3（简短）

景点或场景 + 有无人物 + 一句风格描述

### 三、互动数据 + 为什么高

写出数据，并分析评论占比高的原因（1-2条）：
- 哪个痛点让读者忍不住评论
- 哪个钩子在引导咨询

### 四、目标读者角色

从以下角色中判断这篇笔记主要命中哪一个（可多选）：

| 角色 | 核心顾虑 |
|------|---------|
| 带父母出行的子女 | 老人体力/语言/出行强度，想省心 |
| 情侣/夫妻 | 想深度旅行，不想走马观花，预算有限 |
| 多人家族出行 | 人多难协调，想专属不受干扰 |
| 带孩子的家庭 | 孩子体验 + 行程不能太累 |
| 攻略做崩的人 | 越查越乱，想把决策权交出去 |
| 第一次出国 | 语言不通，怕被坑，没经验 |

写出：这篇主要命中 [角色]，因为正文中 [具体依据]。

### 五、正文场景提取

只提取**具体的服务细节或客人行为**，跳过泛化表达。

- ✅ 提取：「导游随身带折叠小凳，看老人累了就拿出来」
- ❌ 跳过：「全程贴心服务」「行程很舒适」

### 六、笔记结构拆解

分析正文的组织方式，不评价好坏，只描述它是怎么写的：

- **开头方式**：属于哪种类型（成果先说 / 顾虑先说 / 澄清非广 / 做攻略崩溃 / 圆梦型）
- **中间展开**：按什么顺序写的（决策过程 → 具体细节 → 效果验证？还是其他？）
- **段落特点**：长段还是短段、有没有用序号/emoji分段
- **结尾方式**：家人变化收尾 / 直接推荐 / 问句钩子 / 其他
- **整体节奏**：几个转折点，情绪弧线怎么走的

### 七、标题结构分析

拆解标题用了什么公式，关键词前置的逻辑，为什么有效。

示例：`带68岁父母法意瑞，人均4W定制游，回来了`
→ 公式：[对象年龄+关系] + [目的地] + [价格] + [服务类型] + [完成动作]
→ 有效原因：数字具体（68岁/4W）、"回来了"表示可复制、引发好奇

### 八、金句提取（角色代入法）

先代入「四、目标读者角色」中确定的角色，用那个人的眼光重新读一遍正文，找出让他停下来的句子。

判断标准：
1. **说出了读者心里有但没表达的话**（共鸣型）
2. **用具体细节替代了模糊感受**（可信型）
3. **有意外感或反差**（记忆型）

每条金句标注触达的角色和原因：
> 「以前总觉得他们不爱旅游，现在才懂——不是不喜欢，是怕给我们添麻烦」
> → 触达：带父母出行的子女｜原因：说出了很多子女心里压着的愧疚感，读到会停

### 九、AI 联想 → 服务场景

基于图片或正文中实际观察到的内容，按以下格式输出：

**[图片/场景描述]**
用户顾虑：「[用目标读者的口吻写出他们的担心]」

服务场景：[我们具体怎么解决——写成具体服务安排，不是泛化描述。不确定的写「[待补充]」]

### 十、结构匹配 → 模板判断

读取 `笔记模板库/小团笔记模板.md` 和 `笔记模板库/仿写模板库.md`，分两步判断。

#### 10-A：与现有模板对比

将「六、笔记结构拆解」的结果与 A/A2/B/C/D/E 型逐一对比。

**匹配到现有模板时**：
> 此笔记结构匹配模板：**[模板名称，如：模板A·决策型叙事流]**
> 匹配依据：[开头/展开/结尾哪些要素吻合]
> 内容按正常流程归库，模板库无需更新。

**匹配到 E 型时**，额外标注：
> ⚠️ E型为服务维度公式化结构，写稿时优先参考仿写模板库中对应人群的有机参考笔记，而非直接套用E型槽位。

**未匹配到现有模板时**，判断是否值得新建，依据以下任一条件：
1. 用户备注中明确含「模板」字样 → **直接新建**
2. 用户备注含正面评价（「可以参考」「不错」等）→ **询问用户**是否新建
3. 此结构已在素材库中出现2篇以上（同行重复用说明结构有效）→ **询问用户**是否新建
4. 评论占比≥30% 且结构清晰 → **询问用户**是否新建

**新建模板时**，追加到 `笔记模板库/小团笔记模板.md`，格式与现有模板一致：
- 模板命名（如：模板D）+ 数据来源说明
- **骨架**：锚点表格（锚点 / 功能）
- **各受众版本**（带父母版 / 蜜月版 / 带孩子版，按实际适用填写）：开头 → 核心内容 → 收尾杀手锏 → 结尾
- **数据表现**：总互动、评论占比
- 同步在文件顶部「模板类型概览」表格中新增一行

#### 10-B：仿写模板库判断

对照 `笔记模板库/仿写模板库.md`，逐一检查以下三种情况：

**情况一：与现有仿写参考高度相似**
> 此笔记结构接近仿写模板库「参考X·[标题]」，可作为该参考的补充版本。无需再次入库。

**情况二：符合入库标准且填补空白**（同时满足下方三条）
- 评论占比 ≥ 30%
- 结构有机（维度来自真实担忧，不是预设服务分类）
- 有至少1-2个具体场景（位置锚点 + 人物行为反应）

且覆盖的出行类型/视角在库中尚未有对应参考（或现有参考偏少）→ **询问用户**是否追加入仿写模板库

**情况三：不符合入库标准**
> 此笔记为公式化结构（分点/清单型）/ 评论占比低于30% / 缺乏具体场景，不建议入仿写模板库。
