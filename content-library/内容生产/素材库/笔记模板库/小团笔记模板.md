## 模板 A：决策型·叙事流

> 来源：亚宁/20251112-8家大实话，总互动140，评论占比56.4%

### 骨架

| 锚点 | 功能 |
|------|------|
| 开头钩子 | 点出选团之难，制造共鸣 |
| 淘汰过程 | 具体说为什么不选，建立信任 |
| 转折·测靠谱度 | 用一个具体问题测出靠谱度 |
| 服务细节 | 证明选对了 |
| 价格明说 | 筛选有意向读者 |
| 结尾钩子 | 引导咨询/评论 |

### 带父母版

- **开头**：调用开头库（崩溃型 / 找团难型）
- **淘汰**：大团（行程赶/大巴挤陌生人）/ 自由行（老人体力语言搞不定）/ 低价小团（签证退款含糊/成团人数不定）
- **测靠谱度**：「老人腰不好，全程坐车时间怎么控制？」能给出具体方案的留下，含糊的淘汰
- **服务细节**：调用场景库 → 老人专项场景（折叠小凳 / 避开桥梁路线 / 商务车真皮座 / 不早起）
- **结尾**：调用结尾库 → 带父母版开放问答型或价值肯定型（不用"评论区扣1"句式）

### A型带父母版·实战经验（2026-04-04 第一次实践）

**1. 淘汰要三层，不是两层**
通用筛选（低价/成团不保障）→ 受众个性化筛选（带父母专属：会不会照顾老人？）→ 核心问题测试（签证/全包/购物）。三层递进，读者跟着走，逻辑清晰。同一层次的淘汰理由不要堆在一起，容易让人感觉重复。

**2. 要有一段正面说"为什么选小团"**
只说大团/自由行不行还不够，要正面点出小团的核心卖点：就这几个人/不跟陌生人/节奏自己说了算。最好有1-2个具体向往场景（比如"想在某地悠闲半天不赶路"），不要只列功能。

**3. 测靠谱度必须写两家的回答**
这段的张力来自对比。要同时写出"差回答长什么样"和"好回答长什么样"，两种回答并排，读者自己判断。光写"这家给出了具体方案"而没有对比，说服力打折。

**4. 服务细节要给司导一个人格**
行程手册、商务车是功能；司导做了什么一件具体的事、说了什么一句话，才是有温度的人格化。下次写带父母版时，在服务细节段单独给司导一个具体行为，不要把人格化细节全藏在小插曲里。

**5. 调研来源写出来**
"在小红书和各大旅游平台对比了X家"一句话，让读者相信这个人认真比较过，不是软文。比光说"对比了X家"可信度高。

### 蜜月情侣版

- **开头**：调用开头库（挑旅行社大实话型 / 找团崩溃型）
- **淘汰**：低价陷阱 / 客服一问三不知（签证退款含糊 / 有强制购物）/ 行程流水线（没有蜜月浪漫感）
- **测靠谱度**：「如果在巴黎赶火车没赶上，后面行程怎么办？」个人定制师含糊 → 旅行社有专属顾问能最快协调
- **服务细节**：调用场景库 → 浪漫加分项（蒙马特法式下午茶 / 佛罗伦萨私藏夜景机位 / 威尼斯贡多拉 / 不早起）
- **结尾**：「想去欧洲蜜月的姐妹直接抄作业」/ 开放问答

### 带孩子版

> 来源：定制自由行/20260227-带娃意大利，评论占比67.3%（本库最高）
> 特点：以「三步筛选法」替代横向淘汰清单，叙事感更强；开头结论先说而非顾虑先说

- **开头**：调用开头库（结论先说型，如「这趟真没累趴，先说结论：带孩子出国，千万别跟团！！」）
- **淘汰**：不问孩子年龄喜好就直接发方案 / 行程比跟团还赶（一天8个景点）/ 无梵蒂冈等热门景点快速通道 / 合同随意说靠口碑
- **测靠谱度**：「我家孩子X岁，喜欢[兴趣方向]，行程怎么安排？」→ 能说出具体方案（多留一天/孩子专属讲解）的留下；只发标准行程表的淘汰
- **服务细节**：调用场景库 → 带孩子专项（斗兽场孩子专属互动讲解 / 梵蒂冈快速通道 / 酒店选景点附近孩子随时可回休息 / 行程中突发情况实时支持）
- **结尾**：调用结尾库 → 带孩子版引导钩子

### 骨架

| 锚点 | 功能 |
|------|------|
| 开头一句定性 | 直接切主题，阅读成本低 |
| 不能选的信号（3-4条）| 帮读者排除错误选项 |
| 选择原因（3-4条）| 正面说清楚为什么选 |
| 价格明说 | 筛选有意向读者 |

### 带父母版

- **不能选的信号**：报价低得离谱 / 签证拒签退款含糊 / 行程有强制购物 / 问老人体力安排回「都会照顾」无具体方案

- **结尾**：「准备带父母去法意瑞的可以收藏，少走弯路」

### 蜜月情侣版

- **不能选的信号**：低价精致小团必有隐藏消费 / 问签证退款含糊 / 行程流水线没有蜜月浪漫感
- **选择原因**：定制小团（小团价格享定制服务）/ 定制师懂少女心 / 不早起 / 自由时间充足
- **结尾**：「想去欧洲蜜月的姐妹直接抄作业！」

### 带孩子版

> 来源：定制自由行/20260227-带娃意大利，评论占比67.3%
> 特点：清单式呈现筛选标准，逻辑性强，干货密度高

- **不能选的信号**：调用淘汰细节库 → 带孩子出行专用（不问喜好/行程太满/无快速通道/合同随意）
- **选择原因**：行程按孩子年龄和兴趣定制（历史迷多排罗马）/ 专为孩子设计的景点讲解方式 / 热门景点提前预约快速通道 / 行程中突发状况有专人实时支持
- **结尾**：「带娃出国想少踩坑的可以收藏」/ 开放问答

| 锚点 | 功能 |
|------|------|

## 模板 D：家庭成员视角型

> 来源：定制自由行/20260225-一家四口法意瑞；定制自由行/20260227-一家四口意瑞已回；亚宁/20260330-一家四口
> 触发理由：同一结构在竞品中重复出现3篇以上；其中有数据的一篇评论占比47.8%
> 适用业务：我们目前无直接对应笔记，可作为带孩子题材的新方向参考

### 骨架

| 锚点 | 功能 |
|------|------|
| 开头（澄清非广）| 「不是托/纯素人分享」，一句话建立普通用户身份 |
| 家庭成员分项（3-4人）| 每人一个最满意的点，✅或1️⃣分段，覆盖全家不同顾虑 |
| 费用明说 | 团费+含机票签证总价，结构清晰 |
| 结尾钩子 | 「有问题评论区说，我看到都会回复」 |

**核心逻辑**：每个家庭成员有不同顾虑（妈妈怕操心/爸爸怕带路/孩子怕无聊/小女儿要作息正常），分项写让每类读者都能找到自己，命中感比泛写「全家都满意」强。

### 带孩子版（一家四口）

- **开头**：「不是托，[身份说明]。刚带一家四口从[目的地]玩了[X]天回来，趁记忆还热乎整理出来」
- **妈妈满意的是**：行程完全按需求定制，签正规合同无购物店，个性化需求全满足
- **老公满意的是**：不用操心地图/交通/酒店，节奏松弛，体力有余反而主动当摄影师
- **儿子满意的是**：定制师预约了适合孩子的专业讲解 + 推荐餐厅精准踩中孩子口味
- **女儿满意的是**：不用早起赶路，行程按正常作息走，孩子情绪稳定
- **费用**：「[X]人，人均团费[X]w，含机票签证人均[X]w，早餐酒店含，午晚餐自理」
- **结尾**：调用结尾库 → 带孩子版引导钩子

## 模板 E：服务维度分项型

> 与C型的区别：C型按体验感受罗列（情感主导），E型按服务类别分项（理性主导，适合做决策的读者）
> **图片角色**：E型不靠图片证明服务维度，图片只证明"我们真的去了这些地方"。每个服务维度靠文字叙事证明——一句功能定义 + 一个具体事件（人物行为/对话/意外反转）。

### 骨架

| 锚点 | 功能 |
|------|------|
| 开头 | 澄清非广 或 成果先说（「和妈妈刚从X回来，分享几个让我放心的点」）|
| 服务维度分项（3-5项）| 每项一个服务类别，1️⃣2️⃣分段，**每项必须有一个具体事件支撑，不能只有功能描述** |
| 选团避坑建议 | 2-3条注意事项，理性收尾 |
| 结尾钩子 | 家人行为变化 + 引导收藏或评论 |

### ⚠️ 业务类型决定维度内容：小团/定制小团 vs 定制自由行

**我们的业务（小团/定制小团）**：司导全程随行 + 商务车，出行中客人不需要自己导航或坐公共交通。

**定制自由行**（竞品/参考笔记来源）：无司导随行，客人自己用App/行程手册导航，坐火车/地铁。

读取定制自由行源笔记时，按以下规则转换，**不能照搬**：

| 定制自由行元素 | 我们的写法 |
|---|---|
| 路书App / 行程手册导航 | **删除**——司导在，客人不需要自己导航 |
| 城市间火车票怎么买/怎么坐 | **删除**——改为商务车转移，不用操心 |
| 24H顾问远程响应（临时现场问题） | **改为**司导现场直接解决 |
| 24H顾问远程响应（后台事务）| **保留**——新增景点门票/增加酒店/换司导等，我们也是定制师来处理 |
| 顾问远程推荐餐厅 | **保留并扩展**——定制师出发前提供推荐清单，司导现场也会推荐 |
| 酒店安排（位置/选址） | 保留 |
| 门票预订 | 保留，改写为「定制师提前订好，到门口直接进」 |

### 带父母版（小团/定制小团）✅ 主用

- **开头**：「和妈妈刚从[目的地]玩回来，[X]天，分享几个真正让我放心的点」
