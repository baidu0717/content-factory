## 本次完成：内容改写 API 从 Gemini 切换到 Claude Sonnet 4.6

### 背景

### 技术方案
- **模型**：`anthropic/claude-sonnet-4.6`（OpenRouter）
- **调用方式**：原生 `fetch`（放弃 OpenAI SDK，避免认证头冲突）
- **路由文件**：`app/api/xiaohongshu/rewrite/route.ts`
- **新增环境变量**：
  - `OPENROUTER_API_KEY`：OpenRouter API Key
  - `REWRITE_MODEL`：模型 ID（默认 `anthropic/claude-sonnet-4.6`）

### 排查过程中遇到的坑

1. **Vercel CLI 添加的 env 不等于 Dashboard 设置的 env**
   - CLI `vercel env ls` 只显示 CLI 添加的变量，Dashboard 手动设置的不在其中
   - 最终需要在 Dashboard 手动添加才生效

2. **OpenAI SDK 不适合 OpenRouter 认证**
   - `new OpenAI({ apiKey, baseURL })` 无法正确传递 Authorization 头给 OpenRouter
   - 加 `defaultHeaders` 也无效
   - 解决：改用原生 `fetch` 手写 `Authorization: Bearer` 头

3. **Vercel Dashboard 手动填写 API Key 时字符输入错误**
   - `o`（字母）被打成 `0`（数字）：`sk-0r-v1-` vs `sk-or-v1-`
   - 通过日志打印前12位发现
