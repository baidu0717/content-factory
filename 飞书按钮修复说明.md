# 🎯 飞书"去复刻"按钮问题已解决

## 问题原因

您的飞书表格中的**正文字段名称是 `正文内容`，不是 `正文`**！

这导致按钮URL中的 `{{正文}}` 无法被飞书替换为实际的字段值，所以您看到的控制台输出是：
```
[URL参数] title: {{标题}}
[URL参数] content: {{正文}}      ← 这里应该是 {{正文内容}}
[URL参数] tags: {{话题标签}}
```

## 解决方案

### 方法1：修改飞书按钮配置（推荐）

在飞书表格中，修改"去复刻"按钮的链接地址为：

```
https://content-factory-jade-nine.vercel.app/rewrite?title={{标题}}&content={{正文内容}}&tags={{话题标签}}
```

**关键修改**：将 `{{正文}}` 改为 `{{正文内容}}`

### 方法2：重命名飞书字段（不推荐）

如果您想使用 `{{正文}}`，需要在飞书表格中：
1. 找到"正文内容"列
2. 右键点击列名 → 重命名
3. 改为"正文"

⚠️ **不推荐这种方法**，因为可能会影响已有的API集成代码。

## 修改步骤详解

### 1. 打开飞书多维表格

### 2. 找到"去复刻"按钮字段

### 3. 点击列名右侧的下拉箭头 → "编辑字段"

### 4. 修改链接地址

**修改前**：
```
https://content-factory-jade-nine.vercel.app/rewrite?title={{标题}}&content={{正文}}&tags={{话题标签}}
```

**修改后**：
```
https://content-factory-jade-nine.vercel.app/rewrite?title={{标题}}&content={{正文内容}}&tags={{话题标签}}
```

### 5. 保存配置

## 验证测试

修改后，点击任意笔记的"去复刻"按钮，应该看到：

✅ 浏览器正确跳转到复刻页面
✅ 标题框已预填充
✅ 正文框已预填充
✅ 话题标签框已预填充

浏览器控制台应该显示：
```

[小红书复刻] 从URL参数加载笔记数据

```
